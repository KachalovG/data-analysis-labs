   - одного количественного признака;
   - одного категориального признака.
4. Указать, какие способы обработки пропусков использованы.

## Что сделано

В работе:

1. Загружен набор данных `iris`.
2. Добавлен искусственный категориальный признак `petal_length_group` на основе признака `petal length (cm)`:
   - `short`
   - `medium`
   - `long`
3. Искусственно добавлены пропуски:
   - в количественный признак `sepal width (cm)`;
   - в категориальный признак `petal_length_group`.
4. Для обработки пропусков применены:
   - `median` для количественного признака;
   - `most_frequent` для категориального признака.
5. Построена гистограмма признака `petal length (cm)`.
6. Сохранены таблицы и графики для отчёта.

## Какие признаки использовать для дальнейшего ML

Для построения моделей машинного обучения рекомендуется использовать исходные количественные признаки:

- `sepal length (cm)`
- `sepal width (cm)`
- `petal length (cm)`
- `petal width (cm)`

Почему:

- это исходные измеряемые признаки набора данных;
- они содержат максимум исходной информации;
- они не являются искусственно агрегированными;
- искусственный категориальный признак `petal_length_group` построен на основе `petal length (cm)` и дублирует его информацию.

Следовательно, `petal_length_group` можно использовать только как вспомогательный признак для экспериментов, но в основной модели лучше не включать его одновременно с `petal length (cm)`.

## Использованные способы обработки пропусков

- Количественный признак `sepal width (cm)`:
  использована замена пропусков медианой (`median`).
- Категориальный признак `petal_length_group`:
  использована замена наиболее частым значением (`most_frequent`).

Эти способы выбраны потому, что:

- медиана устойчива к выбросам и хорошо подходит для числовых признаков;
- наиболее частое значение является стандартным и интерпретируемым способом заполнения категориальных пропусков.
